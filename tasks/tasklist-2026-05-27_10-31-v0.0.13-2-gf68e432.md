# GPX Upload & Garmin Connect Import — 2026-05-27_10-31 — v0.0.13-2-gf68e432

> API + UI for: (1) upload a GPX file with attributes, (2) import a track from Garmin Connect by activity ID.

---

## Proposed design

### GPX upload — `POST /api/tracks/upload`
- `multipart/form-data`: `file` (GPX), `trackName` (string), `activityType` (string, optional), `notes` (string, optional)
- Store raw GPX bytes in `raw_payload` column (already exists)
- `source = "gpx-upload"`, `sourceId = null`
- Returns `TrackSummary`

### Garmin Connect import — `POST /api/tracks/import/garmin`
- JSON body: `{ "activityId": "12345678" }`
- Backend fetches GPX from Garmin Connect export URL, stores it, extracts `trackName` from GPX `<name>` element if not overridden
- `source = "garmin"`, `sourceId = activityId`
- Returns `TrackSummary`

**Garmin auth**: Garmin Connect requires a session cookie to download GPX. The two practical options are:
- **A (proposed)**: store Garmin username + password in app config / k8s secret; backend performs a one-shot login on startup or per-request to obtain a session cookie, then uses it to download
- **B**: user downloads GPX manually from Garmin and uploads via option 1 (no Garmin auth needed)

→ **see unresolved questions below before implementation starts**

### UI additions (in `Profile.tsx` or a new `UploadPanel.tsx`)
- GPX upload form: file picker + trackName field + optional activityType + submit button
- Garmin import form: activityId text field + submit button
- Both refresh the track list on success, show an inline error on failure

### 2 — Garmin Connect import API *(depends on answer to Q1)*
- [ ] `GarminConnectClient` service: authenticate + download GPX bytes for an activity ID
  - Config: `garmin.username` / `garmin.password` (injected from env / k8s secret)
  - Auth: POST to Garmin login, capture session cookie, reuse until expired
  - Download: `GET https://connect.garmin.com/modern/proxy/download-service/export/gpx/activity/{id}`
- [ ] `POST /api/tracks/import/garmin` in `TrackController`
  - Body: `GarminImportRequest(String activityId)`
  - Calls `GarminConnectClient`, extracts `<name>` from GPX XML for `trackName`
  - Saves track: `source="garmin"`, `sourceId=activityId`
  - Returns 200 `TrackSummary`, 502 if Garmin unreachable, 409 if `sourceId` already imported
- [ ] `application-local.properties`: add `garmin.username=` / `garmin.password=` (empty → Garmin import disabled locally)
- [ ] `GarminImportIT`: mock `GarminConnectClient` → happy path + duplicate prevention

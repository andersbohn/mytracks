# Task List: Google SSO Open Registration Endpoint
> 2026-05-25_09-16 | v0.0.4-1-ge773d95

Request: API endpoint where an unauthenticated user via Google SSO can register/create themselves as a user entity for further use. To back a React webapp landing page.

---

## Design summary

Registration IS the OAuth2 login flow — `CustomOAuth2UserService.findOrRegister()` already creates the user on first login. Gaps:
- No public endpoint for the React SPA to check auth state or get the login URL
- No CORS config
- No configurable redirect-to-frontend after OAuth2 success

New flow:
1. React calls `GET /api/auth/status` (public) → `{authenticated: false, loginUrl: "/oauth2/authorization/google"}`
2. User clicks "Login with Google" → Google OAuth2 dance → Spring callback → user created
3. Spring redirects to `app.frontend-url` (configurable property)
4. React calls `GET /api/auth/status` → `{authenticated: true, user: {id, email, displayName}}`

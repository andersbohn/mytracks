## Architecture after this task

Three authentication paths, evaluated in filter order:

```
Request
  │
  ├─ Authorization: Bearer <google-id-token>?
  │     → GoogleBearerTokenFilter
  │         validates via HttpGoogleIdTokenVerifier (tokeninfo endpoint)
  │         findOrRegister user
  │         set auth in SecurityContext — NO session created (stateless)
  │
  ├─ auth.mock-email set (local dev only)?
  │     → MockAuthFilter
  │         auto-authenticates as configured email
  │         sets auth in SecurityContext — session-based
  │
  └─ Session cookie (JSESSIONID)?
        → Spring Security session filter (existing)
            OR OAuth2 redirect flow (/oauth2/authorization/google)
            OR POST /api/auth/google  (Google ID token → session)
```

Public endpoints (no auth): `/actuator/health/**`, `/api/auth/status`, `/api/auth/google`

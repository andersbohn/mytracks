# Task List: Google SSO REST Registration Endpoint
> 2026-05-25_11-18 | v0.0.6

Request: Explicit REST endpoint `POST /api/auth/google` where React SPA POSTs a Google ID token credential, backend verifies it, inserts user via UserRepository, sets Spring Security session, returns UserResponse. Supplements (does not replace) existing redirect-based OAuth2 flow.

---

## Design summary

React (Google Identity Services button) → gets `credential` (ID token) → `POST /api/auth/google {credential}` → backend verifies via Google tokeninfo API → `userAuthService.findOrRegister()` → `userRepository.save()` → set auth in SecurityContext → return `UserResponse`.

Token verification: `GET https://oauth2.googleapis.com/oauth2/v3/tokeninfo?id_token={token}` — no extra deps.

---

## Decisions

1. Verification via Google tokeninfo HTTP endpoint — no extra library deps
2. `GoogleIdTokenVerifier` is an interface — `@MockBean` in tests, no WireMock needed
3. Session auth: use `HttpSessionSecurityContextRepository` to persist auth into session so subsequent calls to `/api/auth/status` or `/api/me` work
4. Existing redirect-based OAuth2 flow unchanged
